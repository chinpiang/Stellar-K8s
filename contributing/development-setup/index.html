    Development Setup¶

    Learn how to set up a local development environment for building and testing Stellar-K8s documentation.

    Prerequisites¶

    • Python 3.9 or later
    • pip (Python package manager)
    • Git

    Installation¶

    1. Clone the Repository¶

    git clone https://github.com/OtowoOrg/Stellar-K8s.git
cd Stellar-K8s

    2. Install Dependencies¶

    pip install -r requirements.txt

    3. Verify Installation¶

    mkdocs --version

    Local Development Server¶

    Start the Development Server¶

    mkdocs serve

    The documentation will be available at http://127.0.0.1:8000

    The server includes: - Live reload - changes are reflected immediately - Search functionality - All theme features enabled

    Custom Port¶

    mkdocs serve --dev-addr=127.0.0.1:8080

    Building Documentation¶

    Build Static Site¶

    mkdocs build

    Output will be in the site/ directory.

    Clean Build¶

    mkdocs build --clean

    Strict Mode¶

    Build with strict mode to catch warnings:

    mkdocs build --strict

    Project Structure¶

    Stellar-K8s/
├── docs/                       # Documentation source files
│   ├── index.md               # Homepage
│   ├── getting-started/       # Getting started guides
│   ├── deployment-guides/     # Deployment documentation
│   ├── configuration/         # Configuration references
│   ├── tutorials/             # Step-by-step tutorials
│   ├── troubleshooting/       # Troubleshooting guides
│   ├── api-reference/         # API documentation
│   └── contributing/          # Contributing guides
├── mkdocs.yml                 # MkDocs configuration
├── requirements.txt           # Python dependencies
└── site/                      # Built documentation (generated)

    Writing Documentation¶

    Create New Page¶

    1. Create a new Markdown file in the appropriate directory: 

      touch docs/tutorials/my-new-tutorial.md

    2. Add frontmatter (optional): 

      ---
title: My New Tutorial
description: A comprehensive tutorial for doing something
---

    3. Write content using Markdown

    4. Add to navigation in mkdocs.yml: 

      nav:
  - Tutorials:
    - My New Tutorial: tutorials/my-new-tutorial.md

    Preview Changes¶

    mkdocs serve

    Navigate to your new page to preview.

    Code Examples¶

    Add Syntax Highlighting¶

    Use triple backticks with language identifier:

    ```yaml
apiVersion: stellar.k8s.io/v1alpha1
kind: StellarValidator
```

    Add Filename¶

    Use title attribute:

    ```yaml title="validator.yaml"
apiVersion: stellar.k8s.io/v1alpha1
kind: StellarValidator
```

    Add Line Numbers¶

    ```python linenums="1"
def hello_world():
    print("Hello, World!")
```

    Using Admonitions¶

    Info Box¶

    !!! info "Information"
    This is an informational message.

    Warning Box¶

    !!! warning "Warning"
    This is a warning message.

    Tip Box¶

    !!! tip "Pro Tip"
    This is a helpful tip.

    Collapsible Admonition¶

    ??? note "Click to expand"
    This content is collapsible.

    Testing Documentation¶

    Check for Broken Links¶

    mkdocs build --strict

    Validate Markdown¶

    # Install markdownlint
npm install -g markdownlint-cli

# Run lint
markdownlint docs/

    Spell Check¶

    # Install aspell
sudo apt-get install aspell

# Run spell check
find docs/ -name "*.md" -exec aspell check {} \;

    Troubleshooting¶

    Port Already in Use¶

    # Kill process on port 8000
lsof -ti:8000 | xargs kill -9

# Or use a different port
mkdocs serve --dev-addr=127.0.0.1:8080

    Module Not Found Errors¶

    # Reinstall dependencies
pip install --upgrade -r requirements.txt

    Theme Not Loading¶

    # Clear pip cache
pip cache purge

# Reinstall mkdocs-material
pip install --force-reinstall mkdocs-material
